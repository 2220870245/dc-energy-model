# 子任务 1: 定义建模范围与数据访问方案

## 目标

在正式抽取数据和训练模型前，先冻结项目边界，避免后续样本表、评价指标和标签定义反复变化。

## 核心问题

需要明确：
- 主标签是否固定为 `measured_power_util`
- `production_power_util` 是否作为辅助标签或对照标签
- 样本粒度是否固定为 `PDU x 时间窗`
- 时间窗长度、历史窗口长度、预测步长如何设定
- BigQuery 查询、导出、存储和训练的成本上限是多少

## 执行步骤

### Phase 1: 冻结研究问题
- 定义主任务、辅助任务和非目标
- 冻结样本粒度、时间切分和评价指标
- 形成变量字典：输入特征、标签、切分维度、实验编号规则

### Phase 2: 验证数据访问
- 验证 GCP 项目、BigQuery API、billing 状态
- 验证可以访问 PowerData2019 和 ClusterData2019 的示例查询
- 验证小规模导出流程可行

### Phase 3: 形成实验合同
- 输出一份短文档，写清楚任务边界、成本边界和验收边界
- 为后续 SQL 和训练代码定义统一命名约定

## 阶段性验证

### Gate 1: 范围验证
通过条件：
- 已明确主标签和辅助标签
- 已明确样本粒度、时间窗、预测步长
- 已明确训练集、验证集、测试集切分策略

### Gate 2: 访问验证
通过条件：
- BigQuery 样例查询能成功返回结果
- 已确认数据集访问权限
- 已确认预算和单次查询成本控制方法

### Gate 3: 合同验证
通过条件：
- 已形成可供后续任务引用的范围说明
- 后续任务不再重复讨论主标签和样本粒度

## 交付物
- 范围说明文档
- 数据访问检查清单
- 实验命名和切分约定

## 完成标准
- 后续任务可以直接按照该任务输出开展实现
- 没有阻塞项留给 SQL 或建模阶段
