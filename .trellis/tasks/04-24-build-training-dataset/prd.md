# 子任务 3: 构建训练样本表与数据质量门禁

## 目标

把抽取结果落成可复现的数据集版本，并建立质量门禁，避免模型性能建立在脏数据或时间泄漏上。

## 执行步骤

### Phase 1: 生成数据集
- 将 SQL 抽取结果导出并落地到 `data/raw` 或 `data/interim`
- 聚合为 `PDU x 时间窗` 粒度
- 生成训练所需的数值特征、类别特征和时间特征

### Phase 2: 执行质量检查
- 统计缺失率、重复率、极端值比例
- 检查标签范围、突变点、长尾分布
- 检查是否存在未来信息泄漏

### Phase 3: 冻结数据集版本
- 固定训练集、验证集、测试集切分
- 固定归一化和编码方式
- 输出字段说明和版本号

## 阶段性验证

### Gate 1: 样本有效性验证
通过条件：
- 样本表可以被 pandas 或 parquet reader 正常读取
- 每条样本都能定位到 PDU 和时间窗
- 主标签分布无明显非法值

### Gate 2: 质量验证
通过条件：
- 缺失值、异常值和重复值比例已报告
- 时间泄漏检查通过
- 训练、验证、测试切分不交叉

### Gate 3: 版本验证
通过条件：
- 数据集版本和生成脚本一一对应
- 同一输入可复现同一份训练样本表
- 特征清单和字段解释完整

## 交付物
- 固定版本训练样本表
- 数据质量报告
- 特征清单和切分说明

## 完成标准
- 后续 baseline 和深度学习训练不再直接依赖临时抽取结果
