# 子任务 4: 建立基线模型并完成第一轮实验验证

## 目标

建立一组可靠基线，作为后续深度学习模型是否值得继续投入的判断依据。

## 需要覆盖的基线
- 持久性预测
- 滑动平均
- 线性回归或 Ridge 或 Lasso
- XGBoost 或 LightGBM
- 线性利用率模型与经验非线性模型

## 执行步骤

### Phase 1: 最小基线
- 先实现持久性预测和线性回归
- 固定评价脚本和指标
- 得到第一版可复现实验结果

### Phase 2: 强基线
- 加入树模型和物理启发式模型
- 对比不同特征子集
- 分析误差在不同 PDU 和时间段上的差异

### Phase 3: 发布 benchmark
- 统一结果表结构
- 固定最优 baseline 配置
- 为深度学习模型设定最低超越目标

## 阶段性验证

### Gate 1: 运行验证
通过条件：
- 每个 baseline 都能从命令行启动训练或推理
- 评价指标脚本对所有模型统一生效

### Gate 2: 结果验证
通过条件：
- 至少形成一张统一 benchmark 表
- 误差结果可复现
- 不同模型使用相同的数据切分

### Gate 3: 决策验证
通过条件：
- 已确定最强 baseline
- 已明确深度学习模型需要超越的指标阈值

## 交付物
- baseline 训练脚本
- benchmark 结果表
- baseline 误差分析图

## 完成标准
- benchmark 成为后续所有模型比较的公共基准
