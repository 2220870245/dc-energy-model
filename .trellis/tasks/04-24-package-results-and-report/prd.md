# 子任务 6: 固化复现流程并输出报告材料

## 目标

把前面任务沉淀为一个可交付项目，既能重复运行，也能支撑论文、汇报或答辩材料。

## 执行步骤

### Phase 1: 固化入口
- 统一训练入口、评估入口和推理入口
- 固定配置文件结构、日志路径和模型输出路径
- 写清楚运行顺序和依赖

### Phase 2: 固化评估产物
- 统一保存 benchmark 表、深度学习结果表、误差图和消融图
- 输出关键图表的生成脚本或 notebook
- 统一实验编号

### Phase 3: 报告收口
- 形成项目摘要、方法说明、实验结果、局限性和下一步工作
- 补充可复现运行说明
- 补充已知风险和假设边界

## 阶段性验证

### Gate 1: 复现入口验证
通过条件：
- 新环境按照说明能跑通至少一条完整流程
- 训练、评估、推理入口清晰且不冲突

### Gate 2: 产物验证
通过条件：
- 结果表、图表和模型文件位置固定
- 报告中的数字可追溯到实验输出

### Gate 3: 交付验证
通过条件：
- 有完整运行说明
- 有项目结论和局限性说明
- 可直接用于汇报或进一步写论文

## 交付物
- 可复现运行说明
- 统一入口脚本和配置
- 图表、实验结果与结论文档

## 完成标准
- 项目从实验性代码变成可交付成果
