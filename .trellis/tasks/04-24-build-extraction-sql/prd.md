# 子任务 2: 实现 BigQuery 抽取与关联 SQL

## 目标

建立一套可重复执行的 SQL，用于把 power trace、机器映射和 workload usage 统一到可训练的时间序列样本基础表。

## 输入
- 子任务 1 输出的标签和样本定义
- PowerData2019 表
- ClusterData2019 表
- `machine_to_pdu_mapping`

## 执行步骤

### Phase 1: 写出最小关联链路
- 先抽取单个 cell、少量 PDU、短时间范围的数据
- 实现 power 表抽取
- 实现 machine 到 PDU 的映射
- 实现 instance_usage 与 machine 关联

### Phase 2: 做时间对齐和聚合
- 统一时间粒度
- 解决时间戳对齐、重复记录、空洞窗口问题
- 输出中间表和验证表

### Phase 3: 固化导出接口
- 输出可直接导出为 Parquet 或 CSV 的查询结果
- 固定列名、数据类型和分区策略

## 阶段性验证

### Gate 1: 关联正确性验证
通过条件：
- 能从 power trace 追溯到对应 PDU
- 能从 instance_usage 追溯到对应 machine，再映射到 PDU
- 结果表中主键或唯一键逻辑明确

### Gate 2: 时间对齐验证
通过条件：
- 标签时间和特征时间采用同一窗口定义
- 对齐前后样本数量变化可解释
- 缺失窗口比例已统计

### Gate 3: 成本与导出验证
通过条件：
- 小规模查询成本可接受
- 导出结果能被本地读取
- 结果列与字段说明一致

## 交付物
- SQL 脚本目录
- 样例查询结果
- 数据字典初稿

## 完成标准
- 可以稳定产出中间表和导出表
- 不再依赖手工临时查询
