# 文档语言规范

## 1. 适用范围

本规范适用于项目中的所有描述性文件，包括但不限于：

- `.trellis/spec/**/*.md`
- `.trellis/workspace/**/*.md`
- `reports/**/*.md`
- `sql/README.md`
- `src/**/README.md`
- 任务说明、阶段总结、实验结论、汇报材料

## 2. 强制规则

- 所有描述性内容默认使用中文撰写。
- 允许保留必要的英文代码标识、模型版本名、字段名、路径名、命令名。
- 不允许直接保留整段英文模板作为最终内容。
- 如果脚本自动生成了英文标题、章节名或说明文字，落盘前必须改写为中文。

## 3. 编码要求

- 所有描述性文件统一使用 `UTF-8` 编码。
- 通过脚本写入中文内容时，必须显式指定 `UTF-8`，避免控制台编码导致乱码。
- 如果发现中文乱码，必须在提交前修复，不得带乱码入库。

## 4. 执行要求

- 新增或更新描述性文件时，优先检查标题、摘要、章节名、表头是否为中文。
- 会话日志、任务记录、项目报告、导师汇报材料必须使用中文表述。
- 对外展示的实验结论应优先给出中文说明，英文术语仅作为补充。

## 5. 当前项目约定

- 当前主模型版本名可保留为 `model-v2-flex-last-mean`，但其说明、摘要、结论必须使用中文。
- 后续新增的 Trellis 工作日志、规范文件、报告文件均按本规范执行。
